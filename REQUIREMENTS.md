# 基于 Huffman 编码的文档压缩系统项目要求

本文档整理自 `resources/个人项目-基于Huffman编码的压缩/基于Huffman编码的压缩系统_课程项目说明.pdf`。

## 1. 项目概述

- 项目名称：基于 Huffman 编码的文档压缩系统。
- 适用对象：计算机科学与技术、软件工程及相关专业本科生。
- 建议语言：C++ 推荐，也可使用 Java、Python 等语言。
- 项目性质：综合编程实践项目。
- 总分：100 分。
- 所有任务均为必做。

项目目标是从零实现一个命令行压缩工具：

- 输入普通文件，输出压缩后的 `.huff` 文件。
- 输入 `.huff` 压缩文件，能够还原出原始文件。
- 支持普通压缩、单独解压、自适应流式压缩三种运行模式。

## 2. Huffman 编码基本要求

程序需要实现标准 Huffman 编码流程：

1. 统计频率：扫描文件，统计每个字节 `0x00` 到 `0xFF` 的出现次数，共 256 种字节值。
2. 构建 Huffman 树：将每个出现过的字节作为叶子节点，权重为出现次数；反复合并权重最小的两个节点，直到只剩一个根节点。
3. 生成编码：从根节点出发，左分支记 `0`，右分支记 `1`，到达叶子节点时路径上的 bit 串即为该字节的 Huffman 编码。
4. 压缩与解压：
   - 压缩时，将每个原始字节替换为对应 Huffman 编码并写入压缩文件。
   - 解压时，从 Huffman 树根节点开始，根据 bit 流遍历到叶子节点并还原字节。

### 3.1 UTF-8 处理要求

- 英文在 UTF-8 下与 ASCII 相同，每个字符占 1 字节。
- 中文在 UTF-8 下是变长编码，常用汉字通常占 3 字节。
- 本项目必须按字节处理文件，不需要区分英文字符或中文字符。
- 每个字节都视为 `0` 到 `255` 的数值。
- C/C++ 实现时应使用 `unsigned char` 读取字节，并使用 `count[256]` 一类数组统计频率。
- 文件读写应使用二进制模式，避免文本模式导致换行符或编码处理差异。

参考思路：

```cpp
FILE* fp = fopen("test.txt", "rb");
unsigned char readbuff[1024];
int count[256] = {0};

while (true) {
    int read_size = fread(readbuff, 1, 1024, fp);
    if (read_size == 0) break;
    for (int i = 0; i < read_size; ++i) {
        count[readbuff[i]]++;
    }
}
```

注意：项目提供文件仅用于课程学习，不应传播或用于商业目的。

## 5. 运行模式

### 5.1 模式一：普通压缩

命令格式：

```bash
Huffman -c <文件路径>
```

示例：

```bash
Huffman -c ./data/hamlet.txt
```

程序需要读取指定文件，统计频率，构建 Huffman 树，完成压缩，并在原文件同目录下生成 3 个文件。

| 生成文件 | 命名规则 | 内容说明 |
| --- | --- | --- |
| 压缩文件 | 原文件名 `.huff` | 二进制压缩文件 |
| 频率表 | 原文件名 `_freq.txt` | 文本文件，记录每种字节的出现频率 |
| 解压验证文件 | 原文件名 `_unzip.txt` | 压缩后立刻解压还原，用于验证正确性 |

例如执行：

```bash
Huffman -c ./data/hamlet.txt
```

必须在 `./data/` 下生成：

- `hamlet.huff`
- `hamlet_freq.txt`
- `hamlet_unzip.txt`

### 5.2 模式二：单独解压

命令格式：

```bash
Huffman -x <文件路径>
```

示例：

```bash
Huffman -x ./data/hamlet.huff
```

程序需要读取 `.huff` 压缩文件，解析压缩文件中保存的必要信息，重建 Huffman 树并解码，在压缩文件同目录下生成解压文件。

| 生成文件 | 命名规则 | 内容说明 |
| --- | --- | --- |
| 解压文件 | 原文件名 `_unzip.txt` | 还原后的原始文本 |

例如执行：

```bash
Huffman -x ./data/hamlet.huff
```

必须在 `./data/` 下生成：

- `hamlet_unzip.txt`

### 5.3 模式三：自适应流式压缩

命令格式：

```bash
Huffman -a <目录路径>
```

示例：

```bash
Huffman -a ./data/adaptive_data/
```

该模式模拟数据流逐步到达的场景。程序需要自动处理指定目录下的 `1.txt`、`2.txt`、`3.txt`，按文件名顺序执行三个阶段。每处理一个新文件，都需要累积之前文件的频率并据此更新 Huffman 树。

| 阶段 | 处理文件 | Huffman 树依据 | 生成文件 |
| --- | --- | --- | --- |
| 一 | `1.txt` | `1.txt` 自身频率 | `1_adp.huff`、`1_freq.txt`、`1_unzip.txt` |
| 二 | `2.txt` | `1.txt + 2.txt` 累积频率 | `2_adp.huff`、`2_freq.txt`、`2_unzip.txt` |
| 三 | `3.txt` | `1.txt + 2.txt + 3.txt` 累积频率 | `3_adp.huff`、`3_freq.txt`、`3_unzip.txt` |

一次运行必须完成全部三个阶段，总共生成 9 个文件。

关键要求：

- 阶段二压缩 `2.txt` 时，Huffman 树必须基于 `1.txt + 2.txt` 的累积频率。
- 阶段三压缩 `3.txt` 时，Huffman 树必须基于 `1.txt + 2.txt + 3.txt` 的累积频率。
- 自适应模式下生成的 `_freq.txt` 文件应体现累积频率，而不是当前单个文件的频率。
- `_freq.txt` 格式与普通压缩模式完全一致。

## 7. 功能任务与评分要求

### 7.1 基础压缩与解压，10 分

必须完成：

- 实现 `Huffman -c` 和 `Huffman -x` 两个命令。
- 生成文件的命名和位置严格符合接口规范。
- 压缩再解压后的文件必须与原始文件逐字节完全一致。

`_freq.txt` 频率表格式：

```text
字节值(十进制)    [字符]    出现次数
```

示例：

```text
 97   a   1240
101   e    890
 32   .    560
 10   .    430
...
```

频率表排序规则：

- 按出现次数从高到低排序。
- 出现次数相同的，按字节值从小到大排序。
- 不可打印字符、控制字符、高位字节等，在字符列显示为 `.`。

扣分点：

- 文件名不符合要求，或输出位置不符合要求，此项不得分。

### 7.2 `hamlet.txt` 压缩与优先队列优化，25 分

必须完成：

- 使用 `Huffman -c` 压缩 `hamlet.txt`。
- 确保解压后的 `hamlet_unzip.txt` 与原始 `hamlet.txt` 逐字节完全一致。
- 构建 Huffman 树时，必须使用优先队列或堆来寻找频率最小的两个节点。
- 不能使用每次线性扫描数组寻找两个最小节点的方式。

性能要求：

- 构建 Huffman 树应达到 `O(n log n)` 或与优先队列等价的复杂度。
- C++ 可使用 `std::priority_queue` 配合比较器实现小根堆，也可使用 `std::multiset`。

扣分点：

- 如果没有使用优先队列或堆优化，此项最多得 10 分。

### 7.6 信息熵与编码效率分析，5 分

必须在实验报告中完成：

- 计算每个文件的 Shannon 信息熵。
- 计算每个文件的 Huffman 平均码长。
- 计算每个文件的编码效率。
- 对比 `hamlet.txt` 纯英文文本和 `斗破苍穹.txt` 中英混合文本的编码效率。
- 分析编码效率差异原因，重点说明字节分布差异带来的影响。

公式：

```text
H = -Σ(p_i * log2(p_i))
```

- `H`：信息熵。
- `p_i`：字节 `i` 出现的概率。
- 信息熵是理论上能达到的最短平均码长，即压缩理论极限。

```text
L = Σ(p_i * len(code_i))
```

- `L`：平均码长。
- `code_i`：字节 `i` 的 Huffman 编码。
- `len(code_i)`：该编码的 bit 长度。

```text
η = H / L * 100%
```

- `η`：编码效率。
- 越接近 100%，说明 Huffman 编码越接近理论最优。
